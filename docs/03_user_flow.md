# 03. 사용자 플로우 & 엣지케이스 명세서 (User Flow)

> 각 기능별 **Happy Path → 상태 분기 → Edge Case** 순으로 기술합니다.
> 클로드가 네비게이션 구조(Stack/Tab/Modal)를 짤 때 이 파일을 우선 참조합니다.

---

## 범례

```
[H]  Happy Path   — 사용자가 아무 문제 없이 목적을 달성하는 기본 경로
[E]  Edge Case    — 예외 상황, 오류, 비정상 접근
[S]  State        — 조건에 따른 UI 상태 분기
[A]  Admin Only   — 팀장(관리자) 전용 분기
[N]  Network      — 네트워크·서버 오류 관련
[P]  Permission   — 권한·인증 관련
```

---

## 0. 공통 시스템 플로우

### 0-1. 앱 실행 시 인증 상태 확인

```
앱 실행
  ↓
로컬 Refresh Token 존재?
  ├── YES → 서버에 Access Token 재발급 요청
  │          ├── 성공 → 홈 화면 진입
  │          └── 실패 (토큰 만료·서버 오류) → 로그인 화면으로 이동 + "다시 로그인해주세요" 토스트
  └── NO  → 로그인 화면으로 이동 (온보딩 첫 실행 여부 확인)
```

**Edge Cases:**
- `[E]` 앱 업데이트 후 토큰 구조가 변경된 경우 → 강제 로그아웃 + 재로그인 안내
- `[N]` 앱 실행 시 서버 연결 불가 → 마지막 캐시 데이터로 읽기 전용 모드 진입 + 상단 배너로 "오프라인 상태" 표시
- `[E]` 기기 시간이 서버 시간과 크게 차이 나는 경우 → 토큰 검증 실패 → "기기 시간을 확인해주세요" 안내

### 0-2. 딥링크 (FCM 알림 클릭)

| 알림 유형 | 딥링크 이동 경로 |
|---|---|
| 경기 투표 등록 | /voting → 해당 투표 카드 하이라이트 |
| 리마인드 알림 | /voting → 해당 투표 카드 하이라이트 |
| 경기 시작 30분 전 | /today-match |
| 경기 종료 후 기록 요청 | /today-match (기록 입력 모드) |
| MVP 투표 요청 | /match-record-detail?tab=mvp |
| MVP 결과 공지 | /match-record-detail?tab=mvp |
| 게시글·공지 알림 | /team-board/:postId |
| 댓글 알림 | /team-board/:postId#comment |
| 가입 신청 접수 (팀장) | /my-team?tab=members&section=requests |
| 가입 신청 결과 (신청자) | /my-team 또는 /team-search |

**Edge Cases:**
- `[P]` 비로그인 상태에서 딥링크 클릭 → 로그인 화면 → 로그인 완료 후 원래 딥링크 목적지로 이동 (Deferred Deep Link)
- `[E]` 이미 삭제된 투표·게시글 딥링크 클릭 → "삭제된 항목입니다" 토스트 + 상위 목록 화면 이동
- `[E]` 탈퇴한 팀의 알림 딥링크 → "더 이상 해당 팀의 팀원이 아닙니다" 안내 + 홈 이동

## 1. 인증 플로우

### 1-1. 회원가입 Happy Path

```
[H] 이메일 입력 → 비밀번호 입력 → 비밀번호 확인 → 닉네임 입력
  → '가입하기' 버튼 클릭
  → 서버 중복 이메일 검증
  → 성공 → 자동 로그인 → 프로필 초기 설정 화면 진입 (온보딩)
```

**Edge Cases:**
- `[E]` 이미 가입된 이메일 입력 → "이미 사용 중인 이메일입니다" 인라인 오류
- `[E]` 비밀번호 8자 미만 → "비밀번호는 8자 이상이어야 합니다" 인라인 오류
- `[E]` 비밀번호 ≠ 비밀번호 확인 → "비밀번호가 일치하지 않습니다" 인라인 오류
- `[E]` 닉네임 특수문자 포함 → "닉네임은 한글·영문·숫자만 사용 가능합니다" 인라인 오류
- `[N]` 가입 요청 중 네트워크 끊김 → 로딩 스피너 표시 후 타임아웃 → "네트워크 오류. 다시 시도해주세요" + 재시도 버튼
- `[E]` '가입하기' 버튼 연속 탭 → 중복 요청 방지 (버튼 즉시 비활성화, 첫 요청만 처리)
- `[E]` 가입 도중 앱 백그라운드 전환 후 복귀 → 입력 내용 유지

### 1-2. 로그인 Happy Path

```
[H] 이메일 + 비밀번호 입력 → '로그인' 버튼 클릭
  → 서버 인증 → Access Token + Refresh Token 발급 → 로컬 저장
  → 팀 가입 여부 확인
    ├── 팀 있음 → 홈 화면
    └── 팀 없음 → 홈 화면 (팀 찾기/만들기 유도 배너 표시)
```

**Edge Cases:**
- `[E]` 이메일 미입력 또는 형식 오류 → '로그인' 버튼 비활성화
- `[E]` 비밀번호 오류 → "이메일 또는 비밀번호가 올바르지 않습니다" (보안상 어느 쪽 오류인지 명시 안 함)
- `[E]` 5회 연속 로그인 실패 → "로그인 시도가 너무 많습니다. 10분 후 다시 시도해주세요" + 버튼 비활성화
- `[E]` 탈퇴한 계정으로 로그인 시도 → "탈퇴된 계정입니다. 복구를 원하시면 고객센터에 문의해주세요"
- `[N]` 서버 점검 중 → "서버 점검 중입니다 (예정 시간: OO시~OO시)" 안내 화면 표시
- `[P]` 다른 기기에서 동일 계정 로그인 → 이전 기기의 Refresh Token 무효화 (기기 1개만 유지 정책 시)

## 2. 팀 관리 플로우

### 2-1. 팀 생성 Happy Path

```
[H] 프로필 > '팀 생성' 클릭
  → Step 1: 팀명 입력 + 지역 선택 + 레벨 선택 → 중복 확인 통과 → '다음'
  → Step 2: 팀 로고 업로드(선택) + 소개글 입력 → '다음' 또는 '건너뛰기'
  → Step 3: 팀원 추가 (카톡 리스트 / 직접 입력 / 초대 링크)
  → '팀 생성 완료' → 생성자 자동으로 팀장 설정 → /my-team 이동
```

**Edge Cases:**
- `[E]` 이미 팀에 소속된 상태에서 팀 생성 시도 → "이미 소속된 팀이 있습니다. 팀을 나가야 새 팀을 만들 수 있습니다" 안내
- `[E]` 중복된 팀명 입력 → "이미 사용 중인 팀 이름입니다" 인라인 오류
- `[E]` 팀명에 비속어·금지어 포함 → "사용할 수 없는 이름입니다" 오류
- `[E]` Step 2에서 로고 이미지 용량 초과 (5MB 이상) → "5MB 이하 이미지를 사용해주세요"
- `[E]` Step 3 카카오톡 리스트 붙여넣기 시 형식이 다를 경우 → "인식되지 않는 형식입니다. 직접 입력을 이용해주세요" 안내
- `[E]` 팀 생성 완료 직전 네트워크 끊김 → 중복 팀 생성 방지를 위해 재시도 전 서버에 생성 여부 먼저 확인
- `[E]` 팀 생성 완료 버튼 연속 탭 → 중복 생성 방지 (서버 측 idempotency key 사용)

### 2-2. 팀 가입 신청 Happy Path

```
[H] 팀 찾기 → 팀 선택 → 팀 상세 확인 → '가입 신청' 버튼 클릭
  → 신청 완료 토스트 + 버튼 '신청 중'으로 변경
  → 팀장에게 FCM 알림
  → 팀장: 신청 목록 확인 → 수락 또는 거절
  → 신청자: FCM 알림 수신 (수락/거절 결과)
```

**Edge Cases:**
- `[E]` 가입 신청 후 팀장이 장시간 응답 없음 → 신청자는 '신청 중' 상태 유지, 별도 만료 처리 없음 (팀장이 처리할 때까지 대기)
- `[E]` 팀장이 거절 후 신청자가 동일 팀에 재신청 → 즉시 재신청 가능 (횟수 제한: 3회 거절 시 해당 팀 신청 7일 차단)
- `[E]` 가입 신청 후 해당 팀이 삭제된 경우 → 신청 목록에서 해당 항목 자동 제거 + 신청자 알림
- `[E]` 팀 정원(최대 30명) 초과 → '가입 신청' 버튼 비활성화 + "팀 정원이 가득 찼습니다" 표시
- `[S]` 비공개 팀 → 팀 검색 결과에 노출되지 않음, 초대 링크로만 가입 가능
- `[E]` 이미 가입된 팀에 중복 신청 → "이미 소속된 팀입니다" 안내

### 2-3. 초대 링크 가입 Happy Path

```
[H] 팀장이 초대 링크 생성 → 카카오톡 등으로 공유
  → 수신자: 링크 클릭
  → 앱 설치 여부 확인
    ├── 설치됨  → 앱 실행 → /team-join/:inviteCode
    └── 미설치  → 스토어로 이동 → 설치 후 딥링크로 /team-join/:inviteCode 자동 진입
  → 팀 정보 확인 → 가입하기 → 안내사항 동의 → 가입 완료
```

**Edge Cases:**
- `[E]` 초대 링크 7일 만료 → "만료된 초대 링크입니다. 팀장에게 새 링크를 요청하세요" 안내
- `[E]` 이미 가입된 팀의 초대 링크 접근 → "이미 가입된 팀입니다" + '내 팀 보기' 버튼
- `[E]` 이미 다른 팀에 소속된 상태에서 초대 링크 접근 → "다른 팀에 소속되어 있습니다. 현재 팀을 나가야 가입할 수 있습니다" 안내
- `[E]` 초대 링크로 가입 신청 후 팀장이 거절 → 일반 가입 신청과 동일한 거절 알림 + 재신청 가능
- `[E]` 팀 정원 초과 상태의 초대 링크 → "팀 정원이 가득 찼습니다" 안내 + 팀장에게 문의 버튼
- `[E]` 링크 재생성 시 기존 링크로 접근 → "유효하지 않은 초대 링크입니다" 안내
- `[N]` 스토어 설치 후 딥링크 미작동 → 앱 첫 실행 시 "초대 링크로 오셨나요?" 배너 표시 + 수동 코드 입력 옵션

## 3. 경기 관리 플로우 (핵심 상세 명세)

### 3-1. 경기 투표 생성 Happy Path

```
[A][H] /voting → '투표 만들기' 버튼 클릭
  → 경기 날짜 선택 → 경기 시간 선택 → 집합 시간 입력
  → 장소 검색 (카카오 로컬 API) → 장소 선택
  → 상대팀명 입력 (선택) → 참가비 입력 (선택) → 메모 입력 (선택)
  → FCM 알림 발송 체크 확인
  → '투표 만들기' 버튼 클릭
  → 서버 저장 → 전체 팀원 FCM 발송
  → /voting 이동 → 생성된 투표 카드 상단 표시
```

**Edge Cases:**
- `[E]` 과거 날짜로 경기 생성 시도 → "오늘 이후 날짜만 선택할 수 있습니다" 달력에서 과거 날짜 비활성화
- `[E]` 같은 날짜·시간에 이미 경기 투표가 있는 경우 → "해당 시간에 이미 경기가 등록되어 있습니다. 그래도 추가하시겠습니까?" 확인 다이얼로그
- `[E]` 장소 검색 결과 없음 → "검색 결과가 없습니다. 직접 장소명을 입력해주세요" 직접 입력 모드 전환
- `[N]` 카카오 지도 API 호출 실패 → "지도 검색을 사용할 수 없습니다. 장소명을 직접 입력해주세요"
- `[E]` 투표 만들기 버튼 연속 탭 → 중복 투표 생성 방지 (첫 클릭 후 버튼 즉시 비활성화)
- `[N]` 저장 중 네트워크 끊김 → "저장에 실패했습니다. 다시 시도해주세요" + 입력 데이터 유지 (폼 초기화 안 함)
- `[E]` FCM 알림 발송 실패 (일부 기기) → 서버 측 발송 실패 로그 기록, 사용자에게는 "투표가 등록되었습니다" 정상 표시 (발송 실패는 백엔드에서 재시도 처리)
- `[A]` 팀장이 아닌 팀원이 URL 직접 접근 → '투표 만들기' 버튼 미표시 또는 권한 없음 처리

### 3-2. 경기 투표 참여 Happy Path

```
[H] FCM 알림 수신 → 알림 탭 → /voting 진입 → 해당 투표 카드 확인
  → '참석' / '미정' / '불참' 버튼 중 1개 탭            ← 3-state (yes/maybe/no)
  → 이전 투표가 있으면 해당 카운트 차감 후 새 상태로 카운트 증가
  → 즉시 참석·미정·불참 인원 수 실시간 업데이트
  → 선택한 버튼 강조 + 나머지 버튼 비활성화 스타일
```

**상태 분기:**
```
[S] 투표 상태별 UI (myVote: 'yes' | 'maybe' | 'no' | undefined)
  ├── 미투표(undefined)  → 참석/미정/불참 버튼 동일 가중치 활성화
  ├── 참석(yes)          → '참석' 초록 강조, 나머지 흐림, 재클릭으로 변경 가능
  ├── 미정(maybe)        → '미정' 노란 강조, 나머지 흐림, 재클릭으로 변경 가능
  ├── 불참(no)           → '불참' 빨강 강조, 나머지 흐림, 재클릭으로 변경 가능
  └── 마감(closed)       → 세 버튼 모두 비활성화, 최종 집계 결과만 표시
```

**Edge Cases:**
- `[E]` 투표 마감 시간 직전 동시 투표 (Race Condition) → 서버 측 타임스탬프 기준으로 마감 전 투표만 유효 처리
- `[E]` 참석으로 투표 후 미정 또는 불참으로 변경 → 기존 yes 카운트 -1 → 새 상태 카운트 +1, 서버 단 트랜잭션으로 처리
- `[E]` 투표 변경 후 다시 변경 → 제한 없이 마감(deadline) 전까지 변경 가능
- `[E]` 경기 당일 투표 (당일 투표 허용 여부) → 경기 시작 2시간 전까지만 투표 가능 (설정값, 팀장이 변경 가능)
- `[N]` 투표 버튼 클릭 후 네트워크 끊김 → 낙관적 업데이트(Optimistic Update) 적용 → 서버 응답 실패 시 이전 상태로 롤백 + 오류 토스트
- `[E]` 팀에서 추방된 후 기존 투표 링크 접근 → "해당 팀의 경기 투표에 접근할 수 없습니다" 안내
- `[E]` 팀원이 아닌 용병이 경기 투표를 보는 경우 → 용병에게는 별도 참석 확인 채팅으로 처리 (투표 UI 미노출)
- `[E]` 투표 목록에 경기가 없는 상태 → "예정된 경기 투표가 없습니다" 빈 상태 UI + '팀장에게 경기를 등록해달라고 요청하세요' 안내

### 3-3. 리마인드 알림 발송 Happy Path

```
[A][H] /voting → 투표 카드의 '리마인드 알림' 버튼 탭
  → 현재 미투표자 목록 조회
  → 미투표자 수 확인 다이얼로그: "N명에게 알림을 발송합니다. 발송할까요?"
  → 확인 → FCM 발송 → "N명에게 알림을 발송했습니다" 토스트
```

**Edge Cases:**
- `[E]` 미투표자가 0명인 상태에서 리마인드 버튼 탭 → "모든 팀원이 투표를 완료했습니다!" 안내 다이얼로그 (발송 버튼 없음)
- `[E]` 1시간 내 이미 리마인드를 발송한 경우 → 버튼 비활성화 + "1시간 후 다시 발송할 수 있습니다 (남은 시간: OO분)" 표시
- `[E]` 리마인드 발송 후 즉시 투표한 팀원 → 해당 팀원에게는 알림이 이미 발송됐지만 별도 취소 처리 없음 (알림 수신 후 이미 투표됨을 확인하면 됨)
- `[N]` FCM 서버 오류로 일부 발송 실패 → "OO명 중 OO명에게 발송되었습니다. 나머지는 자동 재시도됩니다" 안내
- `[A]` 팀장이 아닌 팀원 → 리마인드 버튼 미노출

### 3-4. 오늘의 경기 Happy Path

```
[H] 홈 or 내 팀 → 오늘의 경기 카드 탭 → /today-match 진입

[경기 전 모드]
  → 경기 기본 정보 표시
  → 실시간 투표 현황 (참석 N / 불참 N / 미투표 N)
  → 참석자 명단 리스트 표시
  → 포메이션 미리보기 표시 (저장된 포지션 기준)
  → 경기장 위치 지도 미니 카드 표시

[경기 후 모드 — 경기 종료 시간 이후 자동 전환]
  → 기록 입력 UI 표시
```

**경기 전 상태 Edge Cases:**
- `[E]` 포지션 배정이 저장되지 않은 상태 → "포지션 배정이 완료되지 않았습니다" 안내 + '포지션 배정 하러가기' 버튼
- `[E]` 경기장 정보 없음 (미입력) → "경기장 정보가 없습니다" 표시, 지도 카드 미노출
- `[E]` 참석자 0명 상태 → "아직 참석 확정 인원이 없습니다" 표시
- `[S]` 경기 시작 30분 전 자동 FCM 알림 미수신 (앱 포그라운드 상태) → 앱 내 상단 배너로 "30분 후 경기 시작입니다" 표시
- `[E]` QR 코드 생성 후 상대팀 팀원이 없는 경우 → QR 스캔 페이지는 비로그인으로도 접근 가능 (공개 URL)
- `[E]` 여러 경기가 같은 날 등록된 경우 → 가장 빠른 시작 시간 경기를 '오늘의 경기'로 우선 표시, 하단에 다른 경기 카드 노출
- `[A]` 경기 취소 처리 → "경기를 취소하시겠습니까?" 확인 다이얼로그 → 취소 후 투표 참석 팀원 전체에게 FCM 발송

**경기 종료 후 모드 Edge Cases:**
- `[E]` 경기 기록이 이미 입력된 경우 → 기록 입력 모드 아닌 결과 조회 모드로 표시
- `[E]` 팀장이 아닌 팀원이 기록 입력 접근 → "경기 기록은 팀장만 입력할 수 있습니다" 안내 (읽기 전용 표시)
- `[E]` 경기 종료 후 3일 이내 기록 미입력 → 알림 재발송: "아직 경기 기록이 입력되지 않았습니다"
- `[E]` 기록 입력 도중 앱 종료 → 입력 내용 로컬 임시 저장 → 재진입 시 "이전에 입력 중이던 기록이 있습니다. 이어서 작성하시겠습니까?" 다이얼로그

### 3-5. 경기 기록 입력 Happy Path

```
[A][H] /today-match (기록 모드) or /match-record → '기록 추가'
  → 스코어 입력 (우리팀 N : 상대팀 N)
  → 득점자 선택 (참석 선수 목록에서 선택, 멀티 선택 + 골 수 입력)
  → 어시스트 선수 선택 (선택사항)
  → 출전 선수 전체 확인 (참석 선수 기본 선택 + 수동 조정)
  → 경기 통계 입력 (슈팅·점유율 등, 선택사항)
  → '기록 저장' 버튼 탭
  → 서버 저장 → 팀 전적 자동 갱신 → 개인 스탯 갱신 → MVP 투표 FCM 발송
  → /match-record-detail 이동
```

**Edge Cases:**
- `[E]` 스코어 미입력 상태에서 '기록 저장' 시도 → "스코어를 입력해주세요" 인라인 오류 + 저장 버튼 비활성화
- `[E]` 스코어 합이 득점자 수와 불일치 → "우리팀 득점자 선택 수(N명)가 스코어(N골)와 다릅니다. 확인해주세요" 경고 다이얼로그 (저장은 가능, 확인 후 진행)
- `[E]` 동일 선수를 득점자와 어시스트에 중복 선택 → 허용 (자책골·자가어시 상황은 실제로 발생하지 않으므로 UX상 경고만 표시)
- `[E]` 참석 선수 목록 외 외부 용병 선수 기록 → '게스트 선수 추가' 옵션으로 이름만 입력해서 기록 가능
- `[E]` 참가 인원이 0명인 경우 → 경기 기록 입력 불가 + "최소 1명 이상의 참가 인원이 필요합니다"
- `[N]` 저장 중 네트워크 끊김 → 입력 데이터 로컬 임시 저장 → 네트워크 복구 시 자동 재시도 옵션 제공
- `[E]` 기록 저장 버튼 연속 탭 → 중복 저장 방지 (첫 클릭 후 즉시 비활성화)
- `[E]` 경기 기록 수정 시 이미 MVP 투표가 진행 중인 경우 → "MVP 투표가 진행 중입니다. 득점자·출전 선수 수정 시 투표가 초기화될 수 있습니다. 계속하시겠습니까?" 경고

### 3-6. MVP 투표 Happy Path

```
[H] 경기 기록 저장 → 참석 팀원 전체 FCM 발송 ("MVP를 투표해주세요!")
  → FCM 탭 → /match-record-detail?tab=mvp
  → 참석 선수 목록 중 1명 선택
  → '투표 제출' 버튼 탭
  → "투표가 완료되었습니다" 토스트
  → 투표 버튼 비활성화, 내 선택 표시

[종료 조건 중 먼저 도달하는 것]
  → 전체 참석 인원 투표 완료
  → 또는 경기 후 24시간 경과
  → MVP 결과 공지 FCM 발송 + 경기 상세 결과 표시
```

**Edge Cases:**
- `[E]` 경기 참석 인원이 1명인 경우 → 자기 자신에게 투표 불가 → "참석 인원이 부족하여 MVP 투표를 진행할 수 없습니다"
- `[E]` 자기 자신에게 투표 시도 → "자신에게는 투표할 수 없습니다" 안내, 해당 선수 선택 비활성화
- `[E]` 투표 후 마음이 바뀐 경우 → 제출 후 1시간 이내 1회 변경 가능 (1시간 경과 후 잠금)
- `[E]` 24시간 경과 후 미투표자 처리 → 기권 처리 (결과 계산에서 제외), 결과 공지는 정상 발송
- `[E]` 동점 발생 시 → 공동 MVP 처리 (2명 이상 동점 허용), FCM 메시지에 모두 포함
- `[E]` 경기 기록 수정으로 출전 선수가 변경된 경우 → 기존 투표 초기화 + 팀원 전체에게 재투표 요청 FCM 발송
- `[E]` MVP 투표 결과 화면에서 앱을 종료하고 재진입 → 동일 결과 화면 유지 (서버에서 결과 고정 상태 반환)
- `[N]` MVP 결과 FCM 발송 실패 → 서버에서 최대 3회 재시도, 실패 시 로그 기록 (앱 접속 시 배지로 확인 가능)
- `[E]` 투표 참여 후 해당 팀에서 추방된 경우 → 해당 사용자 투표는 유효 처리 (기록 보존), 이후 투표 접근 불가

### 3-7. 팀 전적 통계 갱신 플로우

```
[S] 경기 기록 저장 시 자동 트리거:
  → 스코어 기반 승·무·패 판정 (우리팀 득점 > 상대팀 득점 → 승)
  → 팀 전적 컬렉션 업데이트 (총 경기 +1, 해당 결과 +1, 승률 재계산)
  → 각 출전 선수 개인 스탯 업데이트 (출전 수 +1, 득점·어시스트 반영)
```

**Edge Cases:**
- `[E]` 경기 기록 삭제 시 → 해당 경기 결과만큼 전적 통계 역산(rollback) 처리
- `[E]` 경기 기록 수정 시 (예: 승 → 무로 변경) → 기존 결과 역산 후 새 결과로 재계산
- `[E]` 동시에 여러 기록이 저장되는 경우 → 서버 측 트랜잭션으로 처리 (Race Condition 방지)
- `[E]` 득점자 정보 없이 저장한 경우 (득점 기록 생략) → 팀 전적은 업데이트, 개인 스탯 득점 항목은 갱신 안 됨
- `[E]` 상대팀 이름 미입력 경기 → "상대팀 미상"으로 기록

### 3-8. 경기 기록 수정·삭제 (팀장 전용) Happy Path

```
[A][H] /match-record-detail → '수정' 버튼 탭
  → 수정 가능 항목: 스코어, 득점자, 어시스트, 출전 선수
  → 수정 완료 → '저장' 탭
  → 전적 통계 자동 재계산 → 개인 스탯 재계산
  → 수정 완료 토스트

[A][H] '삭제' 버튼 탭
  → "경기 기록을 삭제하시겠습니까? 모든 스탯이 원복됩니다." 확인 다이얼로그
  → 확인 → 기록 삭제 → 전적 통계·개인 스탯 역산
```

**Edge Cases:**
- `[E]` 수정 저장 중 네트워크 오류 → 기존 데이터 유지, 오류 토스트 + 재시도 버튼
- `[E]` 삭제 후 해당 경기의 MVP 투표 데이터 처리 → 투표 데이터도 함께 삭제
- `[E]` 수정으로 인해 특정 선수 스탯이 음수가 되는 경우 (비정상적 수정) → 최솟값 0으로 클램핑 처리
- `[P]` 일반 팀원이 수정/삭제 API 직접 호출 시도 → 서버 권한 검증으로 403 응답

## 4. 공통 네트워크 오류 처리 정책

| 상황 | 처리 방식 |
|---|---|
| API 호출 타임아웃 (10초 초과) | 로딩 스피너 → "응답이 없습니다. 다시 시도해주세요" + 재시도 버튼 |
| 서버 500 오류 | "서버에 문제가 발생했습니다. 잠시 후 다시 시도해주세요" |
| 서버 503 (점검 중) | 전체 화면 점검 안내 팝업 (예상 완료 시간 표시) |
| 401 (인증 만료) | Access Token 자동 갱신 시도 → 실패 시 로그인 화면 강제 이동 |
| 403 (권한 없음) | "접근 권한이 없습니다" 안내 + 이전 화면으로 이동 |
| 404 (데이터 없음) | "요청하신 정보를 찾을 수 없습니다" 안내 + 상위 목록 이동 |
| 오프라인 상태 | 마지막 캐시 데이터로 읽기 전용 표시 + 상단 배너 "오프라인 상태" |
| 쓰기 작업 중 오프라인 | 로컬 큐에 저장 → 네트워크 복구 시 자동 재시도 |

### 6-2. 용병 매칭 플로우 Happy Path

```
[팀 용병 구함]

  → /mercenary/create-wanted 진입
  → 포지션·날짜·시간·장소·레벨·설명 입력 → '등록' 탭
  → 게시글 생성 → /mercenary 목록 상단 표시

[개인 용병 지원]
[H] /mercenary/wanted/:postId → 게시글 상세 확인 → '지원하기' 버튼 탭
  → 지원 완료 토스트, 버튼 '지원 완료'로 변경
  → 게시글 작성자(팀)에게 FCM 알림

[지원자 확인 — 팀 관리자]
[A][H] /mercenary/applicants/:postId → 지원자 목록 조회
  → 매너 점수 확인 → 수락 또는 거절 처리
  → 채팅으로 연락: /chat/:userId 이동
```

**Edge Cases:**
- `[E]` 이미 지원한 게시글에 재지원 시도 → '지원 완료' 버튼 상태, 추가 지원 불가
- `[E]` 게시글 마감일이 지난 경우 → '지원 마감' 배지 표시, 지원 버튼 비활성화
- `[E]` 게시글 작성자 본인이 자신의 게시글에 접근 → '지원하기' 버튼 미노출, '지원자 보기' 버튼 표시
- `[E]` 용병 게시글 작성 중 앱 종료 → 임시 저장 없음 (입력 초기화), 재작성 필요

### 6-3. 매너 점수 평가 플로우 Happy Path

```
[H] 경기 종료 후 FCM "경기 평가를 남겨보세요" 수신
  → /mercenary/evaluate/:matchId 진입
  → 해당 경기에서 함께 뛴 선수 목록 표시
  → 각 선수: 시간준수 / 실력 / 태도 / 의사소통 별점(1~5) 입력
  → '평가 완료' 버튼 탭
  → 각 선수 매너 점수 서버 업데이트
  → "평가가 완료되었습니다" 토스트 + 목록 이동

[노쇼 신고]
[H] /mercenary/report-noshow/:matchId → 노쇼 선수 선택 → 신고 사유 입력
  → '신고' 탭 → 해당 선수 매너 점수 차감 + 플래그 기록
```

**Edge Cases:**
- `[E]` 평가 가능 기간(경기 후 7일) 경과 → "평가 기간이 만료되었습니다" 안내, 평가 불가
- `[E]` 본인 평가 시도 → 본인 카드 목록에서 제외
- `[E]` 동일 경기 동일 선수 중복 평가 시도 → 서버 409 응답 → "이미 평가를 완료했습니다"
- `[E]` 노쇼 신고 후 상대방이 이의 제기 → 이의 제기 기능은 Phase 3 이후 처리 (현재는 신고 접수 후 운영자 검토)
- `[E]` 매너 점수가 없는 신규 사용자 → 용병 가능 게시글에 매너 배지 미표시 (badge 공란)

### 6-4. 매너 점수 조회 플로우

```
[H] /profile → 매너 점수 섹션 → '자세히 보기' 클릭
  → /mercenary/manner-score/:userId 진입
  → 항목별 점수 (시간준수·실력·태도·의사소통) 레이더/바 차트 표시
  → 최근 리뷰 목록 표시

[H] 용병 가능 게시글에서 MannerScoreBadge 클릭
  → /mercenary/manner-score/:authorId 진입 (동일 화면)
```

**Edge Cases:**
- `[E]` 타인의 매너 점수 조회 → 항목별 점수와 총점만 표시, 개별 리뷰 작성자는 익명 처리
- `[E]` 평가 데이터가 5건 미만 → "아직 평가 데이터가 충분하지 않습니다. N건의 평가가 더 쌓이면 정확한 점수를 볼 수 있습니다" 안내
