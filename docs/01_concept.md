# 01. 서비스 정의서 (Product Concept)

> 이 문서는 프로젝트의 방향성이 흔들릴 때 다시 돌아오는 **북극성** 문서입니다.
> 기능 추가·변경 시 항상 이 문서의 USP와 타겟을 기준으로 판단하세요.

---

## 1. 서비스 개요

| 항목 | 내용 |
|---|---|
| **서비스명** | FC Flow (가칭 — 최종 확정 필요) |
| **한 줄 정의** | 팀 운영을 자동화하는 아마추어 축구팀 올인원 관리 앱 |
| **플랫폼** | React Native (Expo) — iOS / Android 동시 지원 |
| **출시 순서** | iOS 우선 → Android 병행 출시 (미확정, 추후 결정) |

---

## 2. 문제 정의

아마추어 축구팀은 경기 한 번을 위해 **최소 5~6개 앱**을 오가며 행정 업무를 처리해야 한다.

| 업무 | 현재 도구 | 문제 |
|---|---|---|
| 소통·공지 | 카카오톡 | 공지가 채팅에 묻힘, 확인 여부 파악 불가 |
| 참석 투표 | 네이버 밴드 / 카카오톡 | 취합 번거로움, 관리자 수동 집계 |
| 용병 모집 | 네이버 카페 | 외부 플랫폼 이탈, 연락 이중화 |
| 경기 기록·통계 | 엑셀 / 없음 | 누적 관리 불가, 개인 스탯 없음 |
| 회비 정산 | 카카오뱅크 | 납부 현황 실시간 확인 불가 |
| 구장 예약 | 각 시·구청 사이트 | 링크 분산, 주차권 사진 공유 번거로움 |

> **핵심 문제 정의**
> "아마추어 축구팀은 경기 자체보다 운영 행정에 더 많은 에너지를 쓰고 있다."

---

## 3. 핵심 타겟

### Primary User (주 사용자)
- **서울·경기·인천 아마추어 축구·풋살 팀 운영진**
- 나이: 20~40대 남성
- 역할: 팀장(감독), 총무, 주장 — 행정 업무를 직접 처리하는 사람

### Secondary User (보조 사용자)
- **일반 팀원**: 공지 확인, 경기 투표, 개인 스탯 확인 목적으로 사용
- **용병 플레이어**: 팀 없이 개인 자격으로 용병 활동을 원하는 사람

---

## 4. 핵심 가치 (USP)

> **"매주 반복되는 팀 운영 행정을 없앤다."**

| 구분 | 기존 플랫폼 (플랩풋볼, 아이엠그라운드) | FC Flow |
|---|---|---|
| 핵심 초점 | 매칭·예약 중심 | **운영 자동화 중심** |
| 타겟 | 개인 플레이어 | **팀 단위 운영자** |
| 투표 | 없음 | **자동 생성 + 리마인드** |
| 경기 기록 | 없음 | **누적 스탯 + MVP 투표** |
| 용병 | 일부 지원 | **팀 내에서 원클릭 모집** |

---

## 5. 기술 스택 확정

| 영역 | 기술 | 결정 이유 |
|---|---|---|
| **프론트엔드** | React Native (Expo) + TypeScript | 크로스플랫폼, 빠른 개발 |
| **상태 관리** | Redux Toolkit + TanStack Query | 서버 상태 / 클라이언트 상태 분리 |
| **인증 서버** | 자체 서버 구축 (Firebase Auth 미사용) | 직접 제어, 추후 소셜 로그인 확장 유연성 |
| **데이터베이스** | 자체 서버 (Firebase Firestore 병행 검토) | ERD 설계 단계에서 최종 결정 |
| **푸시 알림** | FCM (Firebase Cloud Messaging) | 인증과 별개로 FCM만 사용 가능 |
| **지도** | 카카오 지도 API | 국내 정확도, 로컬 검색 API 제공 |
| **결제** | 아임포트 (iamport) | 용병 참가비 중개 (Phase 3 이후) |

## 6. MVP 출시 전략

### Phase 1 — MVP (현재 개발 대상)
> 핵심 루프: **로그인 → 팀 가입 → 경기 투표 → 경기 기록**

| 카테고리 | 포함 기능 |
|---|---|
| 인증 | 회원가입, 로그인, 로그아웃, 토큰 갱신 |
| 프로필 | 개인 정보 설정, 포지션·등번호 입력 |
| 팀 관리 | 팀 생성, 팀 찾기·가입, 팀원 초대, 팀원 목록, 게시판, 회비 정보, 포지션 배정 |
| 경기 관리 | 경기 투표, 오늘의 경기, 경기 기록 입력·조회 |

### Phase 2
포지션 보드 고도화, FIFA 선수 프로필 카드, 용병 매칭

### Phase 3
구장 지도·예약 연동, 회비 자동 정산, 리그 시스템

### Phase 4
프리미엄 구독, 광고 연동, 영상 분석
