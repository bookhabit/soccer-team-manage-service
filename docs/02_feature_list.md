# 02. 기능 명세서 (Feature List / PRD)

> **우선순위 기준**
> - `P0` — MVP에 반드시 있어야 함. 없으면 앱 출시 불가.
> - `P1` — 있어야 편하지만 없어도 앱은 동작함. Phase 1 후반 또는 Phase 2.
> - `P2` — 있으면 좋은 기능. Phase 2 이후.
>
> **완료 기준(AC)**: 해당 조건을 모두 만족하면 기능 구현 완료로 간주.

## 1. 인증 (Authentication)

### 1-1. 회원가입 `P0`
| 항목 | 내용 |
|---|---|
| 입력 필드 | 이메일, 비밀번호, 비밀번호 확인, 닉네임 |
| 유효성 검사 | 이메일 형식, 비밀번호 8자 이상, 닉네임 2~12자, 중복 이메일 서버 확인 |
| 완료 기준 | 가입 후 자동 로그인 상태 유지, 홈 화면 이동 |

### 1-2. 로그인 `P0`
| 항목 | 내용 |
|---|---|
| 방식 | 이메일 + 비밀번호 |
| 토큰 | Access Token (단기) + Refresh Token (장기) 발급, 기기 로컬 저장 |
| 자동 로그인 | 앱 재실행 시 Refresh Token으로 자동 갱신 |
| 완료 기준 | 로그인 후 이전 화면 상태 복원, 토큰 만료 시 자동 재발급 |

### 1-3. 로그아웃 `P0`
| 항목 | 내용 |
|---|---|
| 동작 | 서버 Refresh Token 무효화 + 로컬 토큰 삭제 |
| 완료 기준 | 로그아웃 후 로그인 페이지로 이동, 이전 데이터 캐시 초기화 |

### 1-4. 비밀번호 찾기 `P1`
| 항목 | 내용 |
|---|---|
| 방식 | 이메일로 임시 비밀번호 발송 또는 재설정 링크 발송 |
| 완료 기준 | 이메일 수신 후 비밀번호 변경 완료, 기존 토큰 전체 무효화 |

### 1-5. 소셜 로그인 `P2`
| 항목 | 내용 |
|---|---|
| 대상 | 카카오, 구글, 애플 (iOS 필수) |
| 완료 기준 | 소셜 계정 연동 후 동일 이메일 기존 계정과 병합 처리 |

## 4. 경기 관리 (Match Management)

### 4-1. 경기 투표 생성 (팀장 전용) `P0`
| 항목 | 내용 |
|---|---|
| 입력 필드 | 경기 날짜, 시간, 집합 시간, 장소(카카오 지도 검색), 상대팀명(선택), 참가비(선택), 메모(선택) |
| 자동 동작 | 생성 즉시 팀원 전체 FCM 알림 발송 (선택 해제 가능) |
| 완료 기준 | 생성된 투표 카드가 /voting 목록 상단에 표시 |

| 완료 기준 | 투표 즉시 참석·미정·불참 인원 수 실시간 업데이트 |

### 4-3. 미투표자 리마인드 (팀장 전용) `P0`
| 항목 | 내용 |

### 4-4. 오늘의 경기 `P0`
| 항목 | 내용 |
|---|---|

| 완료 기준 | 경기 종료 시간 경과 후 기록 입력 UI 자동 표시 |

### 4-5. 경기 기록 입력 `P0`

|---|---|
