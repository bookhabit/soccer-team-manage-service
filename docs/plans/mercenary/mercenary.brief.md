# 용병 기획안

## 개요

두 가지 방향의 게시글이 하나의 용병 탭에서 제공된다.

| 게시글 유형   | 등록 주체     | 목적                    |
| ------------- | ------------- | ----------------------- |
| **용병 구함** | 팀 관리자     | 경기에 필요한 용병 모집 |
| **용병 가능** | 개인 플레이어 | 용병 활동 희망 등록     |

---

## 용병 구함

### 등록 페이지

> 권한: 클럽 관리자(주장·부주장)만 등록 가능

- 팀 정보 (본인 소속팀 자동 채움)
- 포지션 (복수 선택)
- 필요 인원
- 경기 날짜
- 경기 시간
- 경기 장소 (이름 + 주소)
- 실력 레벨
- 참가비 (무료 / 금액)
- 상세 설명 (자유 입력)

### 게시글 수정/삭제

- 등록자(관리자)만 가능
- `OPEN` 상태에서만 수정 가능
- `CLOSED`(마감) / 만료 상태에서는 삭제만 가능

### 게시글 목록 페이지

**게시글 1개 카드 데이터**

- 팀 이름 / 팀 레벨
- 구하는 포지션
- 필요 인원 / 현재 수락 인원
- 경기 날짜 / 시간
- 경기 장소
- 참가비
- 마감 여부

**검색 및 필터**

- 날짜
- 포지션
- 지역
- 실력 레벨

### 게시글 상세 페이지

**팀 정보**

- 팀 이름
- 팀 레벨
- (클릭 시 클럽 프로필 이동)

**구하는 용병 정보**

- 필요 인원
- 포지션

**경기 정보**

- 날짜 / 시간
- 장소 (이름 + 주소)
- 참가비
- 상세 설명

**지원 현황**

- 현재 수락 인원
- 지원하기 버튼 (이미 지원한 경우 "지원 완료" 비활성화, 마감/만료 시 비활성화)

### 지원자 관리 페이지

> 권한: 해당 게시글 등록 팀의 관리자만 접근

**지원자 목록**

지원자 1명 데이터:

- 이름 / 나이 / 프로필 이미지
- 포지션 / 레벨
- 매너 점수
- 용병 경기 횟수
- 지원일시

**수락 / 거절 버튼**

- 수락 시: 양측에 상대방 연락처(이름 + 휴대폰 번호) 인앱 표시, 수락 인원 +1
- **수락 인원이 필요 인원에 도달하면 게시글 상태 자동 `CLOSED`** (추가 지원 불가)
  - CLOSED 시 잔여 PENDING 신청 일괄 자동 거절 + 각 신청자에 거절 알림 (콘솔 로그 + TODO)
- 거절 시: 지원자에게 거절 알림 (콘솔 로그 + TODO)

## 용병 가능

### 등록 페이지

- 선호 포지션 (복수 선택)
- 가능 날짜 (달력, 복수 선택) — **마지막 가능 날짜 경과 시 자동 만료**
- 가능 지역 (복수 선택)
- 가능 시간대
- 자기소개 및 특이사항 (자유 입력)
- 참가비 희망 여부

### 게시글 수정/삭제

- 등록자 본인만 가능
- `OPEN` 상태에서만 수정 가능
- 만료 후에는 삭제만 가능

### 게시글 목록 페이지

**게시글 1개 카드 데이터**

- 프로필 이미지 (User.avatarUrl 사용, v1 AI 생성 제외)
- 이름
- 포지션
- 레벨
- 가능 지역
- 가능 날짜

**검색 및 필터**

- 포지션
- 날짜
- 지역
- 레벨

### 게시글 상세 페이지

**개인 정보**

- 이름
- 나이
- 프로필 이미지 (User.avatarUrl 사용, v1 AI 생성 제외)
- 레벨

**용병 가능 정보**

- 가능 날짜
- 가능 지역
- 가능 시간대

**자기소개**

**플레이어 정보**

- 평점 (매너 점수)
- 경력 (용병 경기 횟수)
- 실력 (초급 / 중급 / 고급)

**영입 신청 버튼**

- 클럽 관리자에게만 표시
- 동일 관리자(팀)가 동일 게시글에 중복 영입 신청 불가 (서버 unique 제약)
- 클릭 시 → 영입 신청 플로우

## 매칭 중개 플로우

### 신청 방향

| 방향      | 주체          | 기능명        | 진입점                            |
| --------- | ------------- | ------------- | --------------------------------- |
| 팀 → 개인 | 클럽 관리자   | **영입 신청** | 용병 가능 상세의 "영입 신청" 버튼 |
| 개인 → 팀 | 개인 플레이어 | **입단 신청** | 용병 구함 상세의 "지원하기" 버튼  |

### 공통: 전화번호 사전 확인 (진입 가드)

- 영입 신청 / 지원하기 버튼 클릭 시 `user.phone` 설정 여부 확인
- 미설정 시 AlertDialog: "연락처를 먼저 설정해주세요" → [나중에] [프로필 설정 가기]
- 설정 완료 후 신청 플로우 진입

### 영입 신청 (팀 → 개인)

1. 관리자가 "영입 신청" 버튼 클릭
2. 신청 메시지 입력 (선택, 최대 100자)
3. 서버에서 개인 플레이어에게 알림 발송 (콘솔 로그 + TODO)
   - 팀 이름 / 경기 날짜 / 경기 장소 / 참가비 포함
4. 개인 플레이어: 수락 / 거절
5. **수락 시**: 양측에 상대방 연락처(이름 + 휴대폰 번호) 인앱 화면 표시
6. **거절 시**: 관리자에게 거절 알림 (콘솔 로그 + TODO)

### 입단 신청 (개인 → 팀)

1. 플레이어가 "지원하기" 버튼 클릭
2. 지원 메시지 입력 (선택, 최대 100자)
3. 서버에서 팀 관리자에게 알림 발송 (콘솔 로그 + TODO)
   - 지원자 이름 / 포지션 / 레벨 / 매너 점수 포함
4. 관리자: 지원자 관리 페이지에서 수락 / 거절
5. **수락 시**: 양측에 상대방 연락처(이름 + 휴대폰 번호) 인앱 화면 표시
   - 수락 인원 +1, 필요 인원 달성 시 게시글 자동 CLOSED + 잔여 PENDING 일괄 거절
6. **거절 시**: 지원자에게 거절 알림 (콘솔 로그 + TODO)

### 연락처 노출 정책

- 수락 확정 전까지 양측 휴대폰 번호 미노출
- 수락 이후 **인앱 화면에서 연락처 직접 표시** (이름 + 휴대폰 번호)
- 별도 인앱 채팅 없음

## 만료 처리

| 게시글 유형 | 만료 조건 | 처리 방식 |
|---|---|---|
| 용병 구함 | 경기 날짜 경과 | 조회 시 동적 계산 (`matchDate < now()`) — DB 상태 변경 없음 |
| 용병 가능 | 마지막 가능 날짜 경과 | 조회 시 동적 계산 (가능 날짜 배열의 `MAX(date) < now()`) — DB 상태 변경 없음 |

- 만료된 게시글은 목록 기본 뷰에서 숨김, 필터로 조회 가능
- 등록자 수동 삭제 언제든 가능

## 블랙리스트 (게시글 노출 제한)

매너 점수·프로필·사용자 정보·경기 정보를 종합한 복합 판단 시스템.  
단순 점수 기준이 아니라 여러 신호를 함께 고려한다.

### 제한 조건 (OR — 하나라도 해당 시 적용)

| 조건                | 기준                                         |
| ------------------- | -------------------------------------------- |
| 매너 점수 극하      | 매너 온도 20°C 이하                          |
| 노쇼 신고 누적      | 승인된 노쇼 신고 **3회 이상** (운영 초기 기준, 추후 조정 가능) |
| 계정 이용 제한 상태 | `UserStatus.RESTRICTED` 상태인 유저          |

### 제한 효과

- 용병 구함 / 용병 가능 게시글 **등록 불가**
- 이미 등록된 게시글은 **목록에서 숨김** 처리 (삭제 아님 — 이력 보존)
- 지원하기 / 영입 신청 버튼 **비활성화**

### 제한 해제

- 매너 점수가 기준치 이상으로 회복되면 자동 해제
- 계정 이용 제한 해제 시 자동 해제
- 노쇼 누적은 **자동 해제 없음** — 운영자 수동 해제만 가능 (추후 관리자 페이지 필요)

## 노쇼 신고 시스템

### 신고 주체 및 대상

| 신고 방향 | 주체 | 대상 | 조건 |
|---|---|---|---|
| 팀이 용병 신고 | 클럽 관리자 | 수락된 용병 (MercenaryApplication.ACCEPTED) | 경기 날짜 이후만 신고 가능 |
| 용병이 팀 신고 | 개인 플레이어 | 수락한 팀 게시글 | 경기 날짜 이후만 신고 가능 |

### 처리 흐름

1. 신고 접수 → `NoShowReport` 생성 (`status: PENDING`)
2. 상대방에게 반박 기회 제공 (알림, 콘솔 로그 + TODO)
3. 운영자가 검토 후 `APPROVED` / `REJECTED` 처리 (추후 관리자 페이지 필요)
4. `APPROVED` 시 해당 유저의 승인된 노쇼 신고 수 +1 → 3회 이상 시 블랙리스트 조건 충족

### NoShowReport 모델

```
id, reporterId, reportedUserId, applicationId (FK → MercenaryApplication),
reason (String, 최대 200자), status (PENDING | APPROVED | REJECTED),
reviewedBy (String?, 운영자 userId), reviewedAt (DateTime?),
createdAt
```

## 확정 사항

| 항목           | 결정 내용                                                               |
| -------------- | ----------------------------------------------------------------------- |
| 탭 구조        | 용병 탭 내 **탑탭**으로 "용병 구함" / "용병 가능" 구분                  |
| 게시글 만료    | 경기(가능) 날짜 경과 시 **동적 계산** — DB 상태 변경 없음. 수동 삭제 병존 |
| 용병 가능 만료 | 가능 날짜 배열 중 **마지막 날짜** 경과 시 만료                          |
| 인원 마감      | 수락 인원이 필요 인원 도달 시 **자동 CLOSED** + 잔여 PENDING 일괄 거절  |
| 연락처 노출    | 수락 확정 후 **인앱 화면에서 표시**                                     |
| 전화번호 가드  | 영입·입단 신청 진입 시 `user.phone` 미설정이면 프로필 설정 유도 AlertDialog |
| 중복 신청 방지 | 동일 게시글에 동일 유저(팀) 중복 신청 **서버 unique 제약**으로 방지     |
| 참가비 중개    | **미구현** — 앱 내 결제 기능 없음                                       |
| 노쇼 신고      | `NoShowReport` 모델 신규 설계. 기준치 3회. 운영자 수동 승인 처리        |
| AI 프로필      | **v1 제외** — `User.avatarUrl` 그대로 사용. 추후 AI 생성 기능 추가      |
| 게시글 수정    | `OPEN` 상태에서만 수정 가능. 마감/만료 후 삭제만 가능                   |
| 알림           | 콘솔 로그 + TODO (알림 기능 구축 시점에 구현)                           |
