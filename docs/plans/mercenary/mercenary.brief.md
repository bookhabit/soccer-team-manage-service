# 용병 기획안

## 개요

두 가지 방향의 게시글이 하나의 용병 탭에서 제공된다.

| 게시글 유형   | 등록 주체     | 목적                    |
| ------------- | ------------- | ----------------------- |
| **용병 구함** | 팀 관리자     | 경기에 필요한 용병 모집 |
| **용병 가능** | 개인 플레이어 | 용병 활동 희망 등록     |

---

## 용병 구함

### 등록 페이지

> 권한: 클럽 관리자(주장·부주장)만 등록 가능

- 팀 정보 (본인 소속팀 자동 채움)
- 포지션 (복수 선택)
- 필요 인원
- 경기 날짜
- 경기 시간
- 경기 장소 (이름 + 주소)
- 실력 레벨
- 참가비 (무료 / 금액)
- 상세 설명 (자유 입력)

### 게시글 목록 페이지

**게시글 1개 카드 데이터**

- 팀 이름 / 팀 레벨
- 구하는 포지션
- 필요 인원 / 현재 지원자 수
- 경기 날짜 / 시간
- 경기 장소
- 참가비
- 마감 여부

**검색 및 필터**

- 날짜
- 포지션
- 지역
- 실력 레벨

### 게시글 상세 페이지

**팀 정보**

- 팀 이름
- 팀 레벨
- (클릭 시 클럽 프로필 이동)

**구하는 용병 정보**

- 필요 인원
- 포지션

**경기 정보**

- 날짜 / 시간
- 장소 (이름 + 주소)
- 참가비
- 상세 설명

**지원 현황**

- 현재 지원자 수
- 지원하기 버튼 (이미 지원한 경우 "지원 완료" 비활성화)

### 지원자 관리 페이지

> 권한: 해당 게시글 등록 팀의 관리자만 접근

**지원자 목록**

지원자 1명 데이터:

- 이름 / 나이 / 프로필 이미지
- 포지션 / 레벨
- 매너 점수
- 용병 경기 횟수
- 지원일시

**수락 / 거절 버튼**

- 수락 시: 양측에 상대방 연락처(이름 + 휴대폰 번호) 알림 전송, 잔여 인원 차감
- 필요 인원 모두 채워지면 게시글 자동 마감
- 거절 시: 지원자에게 거절 알림

## 용병 가능

### 등록 페이지

- 선호 포지션 (복수 선택)
- 가능 날짜 (달력, 복수 선택)
- 가능 지역 (복수 선택)
- 가능 시간대
- 자기소개 및 특이사항 (자유 입력)
- 참가비 희망 여부

### 게시글 목록 페이지

**게시글 1개 카드 데이터**

- 프로필 이미지 (AI 생성)
- 이름
- 포지션
- 레벨
- 가능 지역
- 가능 날짜

**검색 및 필터**

- 포지션
- 날짜
- 지역
- 레벨

### 게시글 상세 페이지

**개인 정보**

- 이름
- 나이
- 프로필 이미지 (AI 생성)
- 레벨

**용병 가능 정보**

- 가능 날짜
- 가능 지역
- 가능 시간대

**자기소개**

**플레이어 정보**

- 평점 (매너 점수)
- 경력 (용병 경기 횟수)
- 실력 (초급 / 중급 / 고급)

**영입 신청 버튼**

- 클럽 관리자에게만 표시
- 클릭 시 → 영입 신청 플로우

## 매칭 중개 플로우

### 신청 방향

| 방향      | 주체          | 기능명        | 진입점                            |
| --------- | ------------- | ------------- | --------------------------------- |
| 팀 → 개인 | 클럽 관리자   | **영입 신청** | 용병 가능 상세의 "영입 신청" 버튼 |
| 개인 → 팀 | 개인 플레이어 | **입단 신청** | 용병 구함 상세의 "지원하기" 버튼  |

### 영입 신청 (팀 → 개인)

1. 관리자가 "영입 신청" 버튼 클릭
2. 신청 메시지 입력 (선택)
3. 서버에서 개인 플레이어에게 푸시 알림 발송
   - 팀 이름 / 경기 날짜 / 경기 장소 / 참가비 포함
4. 개인 플레이어: 수락 / 거절
5. **수락 시**: 양측에 상대방 연락처(이름 + 휴대폰 번호) 알림 전송
6. **거절 시**: 관리자에게 거절 알림

### 입단 신청 (개인 → 팀)

1. 플레이어가 "지원하기" 버튼 클릭
2. 지원 메시지 입력 (선택)
3. 서버에서 팀 관리자에게 푸시 알림 발송
   - 지원자 이름 / 포지션 / 레벨 / 매너 점수 포함
4. 관리자: 지원자 관리 페이지에서 수락 / 거절
5. **수락 시**: 양측에 상대방 연락처(이름 + 휴대폰 번호) 알림 전송
6. **거절 시**: 지원자에게 거절 알림

### 연락처 노출 정책

- 수락 확정 전까지 양측 휴대폰 번호 미노출
- 수락 이후 **인앱 화면에서 연락처 직접 표시** (이름 + 휴대폰 번호)
- 별도 인앱 채팅 없음

## 확정 사항

| 항목           | 결정 내용                                                               |
| -------------- | ----------------------------------------------------------------------- |
| 탭 구조        | 용병 탭 내 **탑탭**으로 "용병 구함" / "용병 가능" 구분                  |
| 게시글 만료    | 경기(가능) 날짜 경과 시 **자동 만료** + 등록자 **수동 삭제** 둘 다 지원 |
| 연락처 노출    | 수락 확정 후 **인앱 화면에서 표시**                                     |
| 중복 신청 방지 | 동일 게시글에 동일 유저 중복 지원 **서버 unique 제약**으로 방지         |
| 참가비 중개    | **미구현** — 앱 내 결제 기능 없음                                       |
| 블랙리스트     | 아래 별도 섹션 참고                                                     |

## 블랙리스트 (게시글 노출 제한)

매너 점수·프로필·사용자 정보·경기 정보를 종합한 복합 판단 시스템.  
단순 점수 기준이 아니라 여러 신호를 함께 고려한다.

### 제한 조건 (AND가 아닌 OR — 하나라도 해당 시 적용)

| 조건                | 기준                                         |
| ------------------- | -------------------------------------------- |
| 매너 점수 극하      | 매너 온도 20°C 이하                          |
| 노쇼 신고 누적      | 승인된 노쇼 신고 N회 이상 (기준치 추후 확정) |
| 계정 이용 제한 상태 | 서비스 이용 제한 중인 유저                   |

### 제한 효과

- 용병 구함 / 용병 가능 게시글 **등록 불가**
- 이미 등록된 게시글은 **목록에서 숨김** 처리 (삭제 아님 — 이력 보존)
- 지원하기 / 영입 신청 버튼 **비활성화**

### 판단 데이터 구성

제한 여부는 아래 데이터를 종합해 서버에서 판단한다.

- `MannerScore` — 매너 온도 현재값
- `NoShowReport` — 노쇼 신고 승인 이력
- `User.status` — 계정 이용 제한 상태 필드 (신규 추가 필요)
- `MatchRecord` — 경기 참여 이력 (허수 계정 판별 보조)

### 제한 해제

- 매너 점수가 기준치 이상으로 회복되면 자동 해제
- 계정 이용 제한 해제 시 자동 해제
- 노쇼 누적은 **자동 해제 없음** — 운영자 수동 해제만 가능 (추후 관리자 페이지 필요)
