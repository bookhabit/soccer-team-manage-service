# 클럽 기획안

## 핵심 정책

- **1인 1팀**: 유저는 하나의 팀에만 동시 소속 가능
- **등번호**: `ClubMember` 테이블에 저장, 팀 내 unique 제약
- **클럽 평점**: 소속 팀원 전체 매너 점수의 평균값
  - `Club` 테이블에 `mannerScoreAvg` 컬럼 별도 저장 (캐싱)
  - 재계산 트리거: 팀원 변동(가입·탈퇴·강퇴) **또는 개인 mannerScore 변경** 시
- **역할(Role)**: 주장 / 부주장 (총무는 추후 별도 기획)
- **지역 데이터**: 행정구역 Seed Data 방식 (공공데이터포털 → Prisma seed.ts)

## 가입 신청 페이지

### 신청자 정보 표시 (읽기 전용)

- 이름 / 나이 / 주 포지션 / 거주 위치 / 레벨
- 용병 경험 있는 경우: 매너 점수, 경기 횟수

### 신청 내용 입력

- **자유 텍스트** 1개 필드 (자기소개 / 지원 동기)
- 최대 500자
- **보안 처리 필수** (서버 전역 적용)
  - **XSS 방지**: `sanitize-html` 라이브러리로 HTML 태그 제거/무력화 (Sanitize Interceptor 전역 적용)
  - **SQL Injection 방지**: Prisma 파라미터 바인딩 — 날 쿼리 사용 금지
  - **Helmet** 도입: HTTP 보안 헤더 강화 (XSS, 클릭재킹 방어)
  - **Global ValidationPipe**: `whitelist: true`, `forbidNonWhitelisted: true` — DTO 미정의 필드 거절

### 중복 신청 방지

- 서버 측 `(userId, clubId)` Unique 제약
- 신청 중인 경우 버튼 → **"신청 취소"** 로 변경
- 신청 취소 시 DB에서 삭제

## 클럽 게시판

### 목록 조회

- 탑탭: 전체 / 공지사항 / 문의
- 게시글 데이터: NEW 뱃지 / 유형 / 제목 / 내용 2줄 / 작성자 / 작성일시 / 조회수 / 댓글 수

### 게시글 작성

- 유형: 공지사항(관리자) / 일반 / 문의
- 제목 / 내용 / 상단 고정 스위치 / 팀원 알림 전송 스위치
- **입력 글자 수 제한** (서버 DTO + 클라이언트 유효성 검사 모두 적용)
  - 제목: 최대 100자
  - 본문: 최대 2,000자
  - 댓글: 최대 500자

### 게시글 수정·삭제 권한

- **수정**: 작성자 본인만
- **삭제**: 작성자 본인 + 관리자(주장)

### 게시글 상세 조회

- 게시글 전체 내용
- 댓글 작성 / 목록 조회 (추가 기획 필요)

### 조회수 카운팅

- **Redis 카운터 캐싱** 방식
  - 게시글 조회 시 Redis `INCR post:{id}:views` 증가
  - 주기적으로(또는 일정 임계값 도달 시) DB에 일괄 flush
- DB 직접 실시간 UPDATE 사용 안 함 (고트래픽 게시글 병목 방지)
- **Redis 장애 시 카운터 유실 허용** — 조회수는 정확도보다 가용성 우선

## 팀 삭제 및 해체

> 권한: 주장만 가능

### 해체 플로우

1. 주장이 "팀 해체" 요청
2. **팀원이 주장 1명뿐인 경우**: 동의 절차 없이 즉시 해체 진행
3. **팀원이 2명 이상인 경우**: 전체 팀원에게 동의 요청 알림 발송
   - **모든 팀원 동의** 시 해체 진행
   - 팀원 1명이라도 거절 시 해체 취소
   - **무응답 처리**: 알림 발송 후 **48시간** 내 미응답 시 '자동 거절'로 간주 → 해체 무효
   - 만료 체크 방식: **클라이언트 요청 시점에 서버가 `expiresAt` 기준으로 만료 여부 판단** (Cron Job 미사용)
   - **투표 중 팀원 탈퇴·강퇴 처리**: 투표 진행 중 해당 팀원이 자진 탈퇴하거나 강퇴되면 **자동 동의로 간주** → 나머지 잔류 팀원 전원 동의 시 해체 진행

### 해체 확정 시 연관 데이터 처리

- 해당 팀에 대기 중(Pending)인 **모든 가입 신청 자동 거절**
- Pending 신청자에게 "팀이 해체되어 신청이 취소되었습니다" 알림 발송

### 데이터 보존 방침

- 다른 팀과의 과거 경기 기록 보존
- 상대팀 표시: **(해체한 팀)** 으로 표기
- 팀원 개인 경기 기록(골·어시스트·MOM)은 익명화 유지

## 클럽 초대

### MVP 범위

- **초대 코드**: 관리자가 생성 → 상대방이 앱 내 직접 입력 → 클럽 입장
- 딥링크 / 웹 유니버셜 링크는 **추후 구현**

### 초대 코드 정책

- 유효기간: **7일**
- 만료 후 재발급 가능 (관리자만)
- 코드 엔트로피: 무작위 8자리 이상 (추측 불가 수준)
- **Rate Limit**: 코드 입력 API — IP당 분당 10회 초과 시 429 응답

### 중복 가입 처리

- 이미 가입된 유저가 초대 코드 입력 시 서버에서 전용 에러 코드 반환
- 앱은 **"이미 가입된 팀입니다"** 팝업 표시

### 코드 만료 처리

- 만료된 코드 입력 시 서버에서 전용 에러 코드 반환
- 앱은 **"초대 코드가 만료됐습니다. 팀 관리자에게 재발급을 요청하세요"** 팝업 표시

## 강퇴

> 권한: 주장·부주장

- 팀원 목록 조회 → 1명 선택 → 강퇴 버튼
- 강퇴 이력 `ClubBanRecord` 테이블에 저장
- **강퇴된 유저는 해당 팀 재가입 불가** (서버에서 블랙리스트 체크)

### 강퇴 트랜잭션 순서 (동시성 충돌 방지)

1. 사용자 역할(Role) 제거
2. 멤버십(ClubMember) 삭제
3. ClubBanRecord 생성

> Prisma `$transaction` 으로 원자적 처리 — 역할 제거 먼저 수행하여 권한 고아 상태 방지

## 페이지네이션 정책

> 팀원 목록, 게시판 목록, 가입 신청 목록 등 리스트 조회 전반에 적용

- **방식**: 커서 기반(Cursor-based) 페이지네이션
- **이유**: 무한 스크롤 UX + OFFSET 방식의 대량 데이터 성능 문제 방지
- **응답 형태**: `{ data: [], nextCursor: string | null }`
- **페이지 크기**: 기본 20건
