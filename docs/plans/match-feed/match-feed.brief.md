# 경기 결과 피드 기획안

> **선결 조건**: 경기 기록 기능(`match.brief.md`) 구현 완료 후 진행

---

## 문제 정의

현재 앱은 "내 팀" 중심으로만 구성되어 있어 경기가 없는 날에는 앱을 열 이유가 없다.
다른 팀의 경기 결과를 피드로 노출하면 팀 외부 유저들의 관심을 유입시키고 앱 재방문을 유도할 수 있다.

---

## 목표

- 전체 클럽의 최근 경기 결과를 타임라인 형태로 노출
- 경기가 없는 날에도 앱을 열게 만드는 콘텐츠 루프 생성
- 지역 / 클럽 / 개인 기준 필터로 내게 관련된 경기를 빠르게 탐색
- 상대 전적 탐색의 진입점 제공

---

## 1. 피드 화면

### 위치

- **홈 탭 내 섹션** — 기존 홈 화면에 피드 섹션 추가

### 접근 권한

- **로그인 필수** — 비로그인 상태에서는 홈 진입 자체가 불가하므로 별도 가드 불필요

### 기본 노출 조건

- `isRecordSubmitted = true` — 기록이 등록된 경기만 노출
- `isDeleted = false` — 소프트 삭제 경기 제외
- 경기 유형 필터 미선택 시: LEAGUE + SELF 모두 노출 (필터로 구분)

### 피드 아이템 구성

**LEAGUE 경기**
```
┌─────────────────────────────────────┐
│ [마무리FC]  3 - 1  [카동FC]          │
│ ⭐ MOM: 이현진                        │
│ 서울특별시 은평구 · 어제 14:00        │
│ 경기장: 월드컵공원 풋살장             │
└─────────────────────────────────────┘
```

**SELF 경기 (자체전)**
```
┌─────────────────────────────────────┐
│ [마무리FC] 자체전                     │
│ A팀 2 - 1 B팀                        │
│ 서울특별시 은평구 · 어제 10:00        │
│ 경기장: 은평구민운동장               │
└─────────────────────────────────────┘
```

| 항목 | 내용 | 출처 |
|---|---|---|
| 홈팀 이름 | `Club.name` | Match.clubId → Club |
| 경기 유형 | LEAGUE(vs 상대팀) / SELF(자체전) | Match.type |
| 스코어 | `homeScore : awayScore` | Match |
| 상대팀 이름 | `opponentName` (LEAGUE만) | Match |
| MOM | MomVote 집계 결과 — 없으면 미표시 | MomVote |
| 클럽 소재지 | 홈팀 소속 지역 (시도 + 시군구) | Club.regionId → Region |
| 경기장 | `Match.location` — 표시 전용, 필터 기준 아님 | Match |
| 날짜 | `startAt` 상대 시간 표기 | Match |

## 2. 필터

피드 상단 필터 바에서 복수 조건을 조합할 수 있다.
**지역 필터 기준은 클럽 소재지(`Club.regionId → Region`)** 이며, 경기 실제 장소(`Match.location`)와 무관하다.

### 2-1. 지역 필터 (시도 → 시군구 드릴다운)

| 단계 | 선택 항목 | 설명 |
|---|---|---|
| 1단계 | 시도 (Province) | `Region.name` 기준 — 예: 서울특별시, 경기도 |
| 2단계 | 시군구 (District) | 시도 선택 후 활성화, `Region.sigungu` 기준 — 예: 은평구, 수원시 |
| 미선택 | 전체 (기본) | 전국 경기 전체 노출 |

- 시도만 선택 시 해당 시도 전체 시군구 포함
- 시군구는 시도 선택 없이 단독 선택 불가 — **서버에서 `district`만 있고 `province` 없으면 400 반환**

### 2-2. 경기 유형 필터

| 값 | 설명 |
|---|---|
| 전체 (기본) | LEAGUE + SELF 모두 노출 |
| 매칭전 | `type = LEAGUE`만 노출 |
| 자체전 | `type = SELF`만 노출 |

### 2-3. 내 클럽 경기결과 필터

- 토글 형태 — "내 클럽만 보기"
- 활성화 시 `Match.clubId = 내 클럽 ID` 조건 추가
- 클럽 소속 상태에서만 활성화 (미소속 시 비활성 + "클럽에 가입하면 사용할 수 있어요" 안내)
- 현재 유저는 클럽 1개만 소속 가능 — "내 클럽" 기준은 단일 클럽
- 지역 필터, 유형 필터와 동시 적용 가능

### 2-4. 내가 뛴 경기 필터

- 토글 형태 — "내가 뛴 경기만"
- 활성화 시 `MatchParticipant.userId = 내 userId` 조건 추가
- 모든 로그인 유저에게 활성화 (클럽 소속 여부 무관)
- 내 클럽 필터와 동시 활성화 시: 내 클럽 경기 중 내가 실제 참가한 경기만

### 필터 조합 매트릭스

| 지역 | 유형 | 내 클럽 | 내가 뛴 경기 | 결과 |
|---|---|---|---|---|
| 전체 | 전체 | OFF | OFF | 전국 전체 피드 (기본) |
| 서울 | 전체 | OFF | OFF | 서울 소재 클럽 경기 |
| 전체 | 자체전 | OFF | OFF | 전국 자체전만 |
| 전체 | 전체 | ON | OFF | 내 클럽의 모든 경기 |
| 전체 | 자체전 | ON | OFF | 내 클럽 자체전만 |
| 전체 | 전체 | OFF | ON | 내가 참가한 모든 경기 |
| 전체 | 전체 | ON | ON | 내 클럽 경기 중 내가 뛴 경기 |

## 3. 상세 진입

- 피드 아이템 탭 → 해당 경기의 **공개 결과 상세 화면** (읽기 전용)
  - 스코어 / 득점 타임라인 / MOM / 참여 선수 수
  - 댓글·영상 탭 미노출 (내부 기능)
  - "상대 전적 보기" 버튼 — LEAGUE 경기에만 표시, 두 클럽 간 상대 전적으로 이동

## 4. 빈 상태 UI

| 상황 | 메시지 |
|---|---|
| 전체 피드 결과 없음 | "아직 등록된 경기 결과가 없습니다." |
| 지역 필터 결과 없음 | "해당 지역에 등록된 경기 결과가 없습니다." |
| 내 클럽 필터 (미소속) | 토글 비활성 + "클럽에 가입하면 우리 팀 경기를 확인할 수 있어요." |
| 내가 뛴 경기 필터 결과 없음 | "참가한 경기 기록이 없습니다." |
| 자체전 필터 결과 없음 | "등록된 자체전 기록이 없습니다." |

## 5. API 설계

| Method | Endpoint | 설명 | Auth |
|---|---|---|---|
| GET | `/match-feed` | 피드 목록 (커서 페이지네이션) | 필수 |
| GET | `/match-feed/:matchId` | 단일 경기 공개 결과 상세 | 필수 |

### 쿼리 파라미터 (`/match-feed`)

| 파라미터 | 타입 | 설명 | 검증 |
|---|---|---|---|
| `cursor` | string? | 커서 (matchId 기반) | |
| `limit` | number? | 기본 20, 최대 50 | |
| `province` | string? | 시도 필터 (`Region.name`) | |
| `district` | string? | 시군구 필터 (`Region.sigungu`) | `province` 없으면 400 |
| `type` | `LEAGUE\|SELF`? | 경기 유형 필터 | |
| `myClub` | boolean? | 내 클럽 경기만 | 클럽 미소속 시 무시 |
| `myMatches` | boolean? | 내가 뛴 경기만 | |
| `from` | date? | 시작 날짜 | |
| `to` | date? | 종료 날짜, 최대 범위 6개월 | from~to 초과 시 400 |

---

## 6. 데이터 모델 영향

- **스키마 변경 없음** — 기존 `Match`, `Club`, `Region`, `MomVote`, `MatchGoal`, `MatchParticipant` 조회로 구성 가능
- **인덱스 추가 권장**:
  - `Match(isRecordSubmitted, startAt DESC, isDeleted)` — 피드 기본 정렬
  - `MatchParticipant(userId)` — 내가 뛴 경기 필터
