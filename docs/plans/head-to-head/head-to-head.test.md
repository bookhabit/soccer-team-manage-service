# Head-to-Head (상대 전적) 테스트 가이드

> Seed 실행 순서: `seed:club` → `seed:match` → `seed:match-feed` → `seed:head-to-head`

---

## 🚩 시나리오 1: 피드 상세에서 H2H 진입 후 전적 확인

**[CASE ID: H2H-04-008, H2H-04-020]**

### 준비물

- 마무리FC 주장 (`captain@mamurifc.test`)
- 선행: `seed:match-feed` + `seed:head-to-head` 완료

### 테스트 순서

1. **로그인**: `captain@mamurifc.test`으로 로그인합니다.
2. **피드 진입**: 홈 탭 또는 매칭 탭에서 경기 결과 피드를 엽니다.
3. **상세 진입**: 카동FC와의 LEAGUE 경기 카드를 탭합니다.
4. **버튼 확인**: 상세 화면 하단에 `"상대 전적 보기"` 버튼이 노출되는지 확인합니다.
5. **H2H 진입**: `"상대 전적 보기"` 버튼을 탭합니다.
6. **전적 확인**: H2H 화면에서 요약 카드와 이력 목록을 확인합니다.

### 체크포인트

- [ ] `"상대 전적 보기"` 버튼이 노출되는가?
- [ ] H2H 화면 상단 요약 카드에 `마무리FC (좌) vs 카동FC (우)` 레이아웃인가?
- [ ] 요약: **7승 · 2무 · 3패** 가 표시되는가?
- [ ] 득점 **17골** / 실점 **11골** 이 표시되는가?
- [ ] 이력 목록 첫 10건이 날짜 내림차순으로 표시되는가?

## 🚩 시나리오 5: 권한 차단 — 비소속 유저

**[CASE ID: H2H-01-005, H2H-01-006]**

### 준비물

- 클럽 미소속 유저 (`newbie@test.com`) 또는 카동FC 주장 (`captain@kadongfc.test`)

### 테스트 순서 (비인증)

1. 로그인하지 않은 상태에서 `GET /clubs/{마무리FC_ID}/head-to-head/{카동FC_ID}` 직접 호출.

### 테스트 순서 (비소속)

1. `newbie@test.com`으로 로그인
2. API 직접 호출: `GET /clubs/{마무리FC_ID}/head-to-head/{카동FC_ID}`

### 체크포인트

- [ ] 비인증 → **401** 응답 (`유효하지 않은 토큰`)
- [ ] 비소속 → **403** `H2H_001` 응답 (`해당 클럽 소속이 아닙니다.`)
- [ ] 클라이언트에서는 버튼 자체가 숨겨져 있어 해당 화면 진입 불가
