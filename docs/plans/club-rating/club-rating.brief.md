# 클럽 레이팅 & 순위 시스템 기획안

> **구현 우선순위**: 최후 단계 (match 기록 기능, matching 기능 구현 완료 후 진행)
> **선결 조건**: `match.brief.md` 기반 Match 테이블 + 경기 결과 기록 기능이 DB에 존재해야 함

---

## 문제 정의

팀 레벨을 수동 기입하면, 실력 좋은 팀도 하/하하/최하 등으로 낮춰 입력해 **양학 매칭**이 발생할 수 있음.

---

## 목표

- **팀 레벨 자동 산출**을 도입해 수동 조작을 최소화
- **전체 클럽 순위**를 도입해 신뢰도 있는 랭킹 제공
- **양학 및 어뷰징 방지**를 전제로 설계

---

## 1. 팀 레벨 자동 산출 (전적 기반)

### 현재 상태

- `Club.level` 필드를 관리자가 수동 기입 (ClubLevel enum: BEGINNER / AMATEUR / SEMI_PRO / PRO)

### 변경안: 전체 클럽 비례 레벨링 (Relative Ranking)

매일 자정 배치 스케줄러로 전체 클럽의 ELO Score를 계산한 뒤 백분율로 레벨을 자동 업데이트.

```
신규 ClubLevel (5단계):
  상위  5%: WORLD_CLASS   — 설명이 필요 없는 전국구 탑티어
  5~20%: PROFESSIONAL  — 지역에서 모르는 사람이 없는 강팀
 20~50%: SEMI_PRO      — 팀워크가 완성된 탄탄한 팀
 50~80%: AMATEUR       — 주말마다 꾸준히 발 맞추는 팀
 80~100%: BEGINNER     — 이제 막 시작한 신생 팀
```

**enum 마이그레이션**: 현재 PRO → PROFESSIONAL + WORLD_CLASS로 분리. 기존 PRO 클럽은 일괄 PROFESSIONAL로 이관 후 다음 배치에서 자동 재분류.

---

## 2. 클럽 순위 산출 공식 (HJ-Rating)

### 핵심 원칙

- 단순 승률만으로 순위를 만들면 "2경기 2승 팀이 1등"이 되는 이상 케이스 발생
- 경기 수 신뢰도 가중치 + 상대 강도를 함께 반영하는 ELO 방식 채택

### UNRANKED 구간

- **10경기 미만** 팀은 `UNRANKED` 상태로 표시
- UNRANKED 팀은 전체 순위 및 자동 레벨 산출 대상 제외
- ELO 점수는 1경기부터 계속 누적됨 → 10경기 달성 순간 자동으로 순위표에 노출

### 점수 산출 공식

```
NewScore = CurrentScore + (K × (Result - Expected))

- Result  : 승리(1) / 무승부(0.5) / 패배(0)
- Expected: 팀 간 레벨 차이에 따른 예상 승률 (강팀이 약팀 만날수록 1에 가까움)
- K       : 경기 수가 적을수록 크고, 많아질수록 안정화 (초반 랭킹 진입 속도 조절)
- 초기값  : 신규 클럽 기본 점수 = 1000점
```

### 레벨 갱신 주기

- ELO 점수: 경기 결과 입력 즉시 실시간 반영
- 레벨 재분류 (%기반 등급): **매일 자정 배치**로 전체 재산정
- **레벨 고정**: 레벨 변화(상승·하락 모두) 후 최소 **1주일**간 추가 변동 방지

---

## 3. 등급별 가중치 승점 로직 (양학 방지 핵심)

| 대진 관계 | 승리 점수 | 무승부 | 패배 감점 | 특징 |
|---|---|---|---|---|
| 강팀 도전 (2단계 위) | 기본 × 2.0~3.0 | +5 / +5 | 소폭 감점 (-2) | 하이 리스크 하이 리턴 |
| 호적수 (동급) | 기본 × 1.0 | +5 / +5 | -10 | 진검승부 |
| 약팀 매칭 (2단계 아래) | 기본 × 0.2 | +1 / +1 | **-30 (역배 대참사)** | 양학 방지 핵심 |

> 무승부는 레벨 차이와 무관하게 양 팀 동점 처리. 강팀 상대 무승부는 동급 무승부와 동일 +5, 약팀 상대 무승부는 +1.

---

## 4. 경기 결과 입력 및 검증

### 입력 주체

- 양 팀 관리자(주장·부주장)가 각각 경기 결과를 입력
- 양측 결과가 **일치하면 자동 확정** → ELO 점수 즉시 반영
- **불일치 시**: 결과 미확정 상태 유지. 양 팀 관리자에게 재확인 알림 발송. 일정 기간(72시간) 내 재입력 없으면 무효 처리

### 어뷰징 방지 장치

- **연패 패널티**: 같은 팀에게 단시간 연속 패배 시 점수 반영 차단
- **약팀 패배 대참사**: 2단계 아래 팀에게 지면 -30점 (일부러 지는 어뷰징 구조적 차단)
- **레벨 고정 기간**: 레벨 변화 후 1주일 유지 → 급격한 등락 방지

---

## 5. 홈 화면 랭킹 UI

### 상시 노출

- **전국 랭킹 Top 10**: ELO 점수 기준 전체 순위 (단일 리스트, 레벨 무관)
- **우리 동네 랭킹**: 클럽의 `regionId`(시군구 단위) 동일 범위 기준 순위
- **급상승 팀**: 최근 5경기 승률이 급격히 오른 팀

### 이벤트성 노출

- **[이 주의 대이변]**: 가장 레벨 차이가 큰 매치에서 하위 팀이 승리한 경기 박제
- **[불타는 연승]**: 최근 5연승 이상 팀 노출
- **[우리 동네 깡패]**: 특정 지역 최고 순위 팀 위트있게 노출

---

## 6. 랭킹/산출 방식 공개 페이지

일반 사용자에게 공개되는 페이지.

- 전체 클럽 순위 조회 (ELO 점수 순 단일 리스트)
- **내 클럽 기준 상대적 위치 표시** (예: "전국 347위 / 우리 동네 3위")
- 레이팅 산출 방식 설명 (HJ-Rating 공식, 레벨 구간 기준 안내)

---

## 7. 시뮬레이션 테스트

다양한 시나리오 데이터를 주입해 산출 결과를 검증하는 스크립트 + 리포트 필요.

- 도구: Jest 기반 시뮬레이션 스크립트 (서버 단위 테스트)
- 검증 시나리오:
  - 2경기 2승 팀 vs 20경기 18승 팀 순위 비교
  - 양학 어뷰징 감지 (약팀에게 일부러 지는 시나리오)
  - 연패 패널티 발동 조건
  - 레벨 고정 기간 적용 여부
  - 결과 불일치 시 72시간 경과 무효 처리
