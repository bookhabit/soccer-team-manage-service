# 파일 업로드 기능 테스트 가이드

> Seed 실행 후 앱을 직접 조작하며 아래 시나리오를 순서대로 검증합니다.

---

## 사전 준비

```bash
cd server && npm run seed:file
```

테스트 로그인: **앱 실행 → 하단 탭 홈 → 🧪 테스트 로그인 → "파일 업로드 기능 테스트" 섹션**

---

## 🚩 시나리오 1: 프로필 아바타 신규 업로드

**[CASE ID: FILE-04-011]**

### 준비물

- `captain@mamurifc.test` — avatarUrl=null (seed 실행 시 초기화됨)

### 테스트 순서

1. **로그인**: `마무리FC 주장 — 아바타 신규 업로드` 버튼으로 로그인
2. **화면 이동**: 하단 탭 `[프로필]` → `[프로필 수정]` 버튼 클릭
3. **아바타 선택**: 화면 상단 아바타 이미지(기본 이미지) 터치
4. **갤러리**: 기기 갤러리에서 JPEG/PNG/WebP 이미지 선택
5. **확인 대기**: 업로드 중 아바타 위에 로딩 인디케이터 노출 대기

### 체크포인트

- [ ] 아바타 터치 시 갤러리 picker가 열리는가?
- [ ] 업로드 중 아바타 위에 반투명 오버레이 + `ActivityIndicator`가 표시되는가?
- [ ] 업로드 완료 후 프로필 편집 화면의 아바타가 새 이미지로 즉시 교체되는가?
- [ ] 프로필 탭으로 돌아갔을 때 `PlayerCard`의 아바타도 새 이미지로 반영되는가?

## 🚩 시나리오 2: 프로필 아바타 재업로드 및 삭제

**[CASE ID: FILE-04-012, FILE-04-003]**

### 준비물

- `file-with-avatar@test.com` — 시나리오 1 완료 후 이 계정으로 업로드 후 재업로드

### 테스트 순서

#### 2-1. 재업로드 (기존 이미지 대체)

1. **로그인**: `파일업로드 전용 — 재업로드·삭제 테스트` 버튼으로 로그인
2. **프로필 수정 진입**: 프로필 탭 → 프로필 수정
3. **첫 번째 업로드**: 아바타 터치 → 이미지 A 선택
4. **두 번째 업로드**: 동일한 방법으로 이미지 B 선택

#### 2-2. 삭제 (기본 이미지 복원)

5. **사진 삭제**: `사진 삭제` 텍스트 버튼 클릭

### 체크포인트

- [ ] 이미지 B 업로드 후 이미지 A 경로의 파일이 서버에서 삭제되는가? (`uploads/avatars/` 폴더 확인)
- [ ] 사진 삭제 후 아바타가 기본 이미지(`/upload/defaults/profile.png`)로 복원되는가?
- [ ] `사진 삭제` 버튼은 avatarUrl이 null일 때 노출되지 않는가?

5. **이미지 선택**: 갤러리에서 이미지 선택

- [ ] 클럽 탭 홈 화면의 로고(`AvatarImage`) 가 새 이미지로 교체되는가?
