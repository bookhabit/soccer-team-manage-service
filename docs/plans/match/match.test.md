## 시드 데이터 상태 요약

| 경기 ID                  | 상태                 | 설명                                                                           |
| ------------------------ | -------------------- | ------------------------------------------------------------------------------ |
| `BEFORE_SELF`            | 경기 전 (D-3)        | 자체 경기, 투표 마감 내일, 주장·부주장 ATTEND, 멤버1 ABSENT, 멤버2·3 UNDECIDED |
| `BEFORE_LEAGUE`          | 경기 전 (D-5)        | 리그 경기 vs 카동FC, 포지션 배정(Q1: 4-3-3) 완료                               |
| `BEFORE_DEADLINE_PASSED` | 경기 전, 투표 마감   | 어제 마감, 투표 버튼 비활성화 대상                                             |
| `DURING`                 | 경기 중              | 30분 전 시작, 1시간 후 종료 예정                                               |
| `AFTER_LEAGUE_RECORDED`  | 경기 후, 기록 완료   | 3:1 승, 골·어시 기록 있음, MOM 완료(멤버1 2표·주장 2표 동점), 상대 평가 완료   |
| `AFTER_SELF_NO_RECORD`   | 경기 후, 기록 미입력 | 자체 경기, 기록 없음                                                           |
| `AFTER_MOM_ACTIVE`       | 경기 후, MOM 투표 중 | 2:2 무승부, 주장이 이미 투표(멤버1 선택), 멤버1은 미투표                       |
| `KADONG_BEFORE`          | 경기 전 (D-7)        | 카동FC 소속 경기 — 마무리FC 계정으로 미노출 확인용                             |

## 🚩 시나리오 1: 경기 목록 진입 및 상태 배지 확인

**[CASE ID: MATCH-04-001, MATCH-04-002]**

### 준비물

- 마무리FC 주장 (`captain@mamurifc.test`)

### 테스트 순서

1. **로그인**: `captain@mamurifc.test`으로 로그인합니다.
2. **화면 이동**: 하단 탭의 `[경기]` 메뉴를 클릭합니다.
3. **목록 확인**: 경기 카드 목록이 표시되는지 확인합니다.

### 체크포인트

- [ ] 마무리FC 소속 경기 7개(`BEFORE_SELF` ~ `AFTER_MOM_ACTIVE`)가 노출되는가?
- [ ] `KADONG_BEFORE` 경기는 목록에 **없는가**? (클럽 격리 확인)
- [ ] `BEFORE_SELF`: 배지가 **"투표 진행 중"** 으로 표시되는가?
- [ ] `BEFORE_DEADLINE_PASSED`: 배지가 **"투표 마감"** 으로 표시되는가?
- [ ] `DURING`: 배지가 **"경기 중"** 으로 표시되는가?
- [ ] `AFTER_LEAGUE_RECORDED`: 배지가 **"3:1 승"** 또는 결과 점수로 표시되는가?
- [ ] `AFTER_SELF_NO_RECORD`: 배지가 **"기록 미입력"** 으로 표시되는가?
- [ ] `AFTER_MOM_ACTIVE`: 배지가 **"MOM 투표 중"** 으로 표시되는가?

## 🚩 시나리오 2: 경기 참석 투표 응답 변경

**[CASE ID: MATCH-04-003, MATCH-03-003]**

### 준비물

- 마무리FC 멤버2 (`member2@mamurifc.test`) — `BEFORE_SELF` 경기에서 **UNDECIDED** 상태

### 테스트 순서

1. **로그인**: `member2@mamurifc.test`으로 로그인합니다.
2. **화면 이동**: `[경기]` 탭 → `BEFORE_SELF` 경기 카드를 클릭합니다.
3. **투표 탭 확인**: 경기 상세에서 `[투표]` 탭을 클릭합니다.
4. **투표 응답**: `'참석'` 버튼을 클릭합니다.
5. **응답 변경**: 다시 `'불참'` 버튼을 클릭합니다.

### 체크포인트

- [ ] `'참석'` 클릭 후 버튼 상태가 **활성(선택됨)**으로 바뀌는가?
- [ ] `'불참'` 클릭 후 참석자 수 카운트가 감소하는가?
- [ ] 변경 후 즉시 UI가 반영되는가? (낙관적 업데이트 또는 쿼리 무효화)
- [ ] 응답 변경 완료 토스트 메시지가 노출되는가?

## 🚩 시나리오 6: MOM 투표 — 미투표 멤버 투표 가능 / 이미 투표한 멤버 비활성화

**[CASE ID: MATCH-04-011, MATCH-04-012, MATCH-03-005]**

### 준비물

- 마무리FC 주장 (`captain@mamurifc.test`) — 이미 멤버1에 투표 완료
- 마무리FC 멤버1 (`member1@mamurifc.test`) — 미투표

4. 투표 목록에서 `"마무리FC 주장"` 선택 후 `'투표'` 클릭.

### 테스트 순서 (주장 — 이미 투표 확인)

1. **계정 전환**: `captain@mamurifc.test`으로 재로그인.
2. 동일 경기 → `[MOM 투표]` 탭.

### 체크포인트 (멤버1)

- [ ] MOM 후보 목록이 표시되는가?
- [ ] 투표 버튼이 **활성화** 상태인가?
- [ ] 투표 완료 후 `"투표 완료"` 메시지가 표시되는가?
- [ ] 투표 후 버튼이 **비활성화(이미 투표)** 로 바뀌는가?

### 체크포인트 (주장)

- [ ] `"이미 투표하셨습니다"` 안내가 표시되는가?
- [ ] 투표 대상(`마무리FC 멤버1`)이 선택된 상태로 표시되는가?
