# FC Flow — 서비스 전체 기획서

> 기능 추가·변경 시 항상 이 문서의 USP와 타겟을 기준으로 판단한다.  
> 상세 기획은 `docs/plans/` 내 각 도메인 파일을 참조한다.

---

## 1. 서비스 개요

| 항목           | 내용                                                |
| -------------- | --------------------------------------------------- |
| **서비스명**   | FC Flow (가칭 — 최종 확정 필요)                     |
| **한 줄 정의** | 팀 운영을 자동화하는 아마추어 축구팀 올인원 관리 앱 |
| **플랫폼**     | React Native (Expo) — iOS / Android 동시 지원       |

---

## 2. 문제 정의

아마추어 축구팀은 경기 한 번을 위해 **최소 5~6개 앱**을 오가며 행정 업무를 처리한다.

| 업무      | 현재 도구   | 문제                                    |
| --------- | ----------- | --------------------------------------- |
| 소통·공지 | 카카오톡    | 공지가 채팅에 묻힘, 확인 여부 파악 불가 |
| 참석 투표 | 네이버 밴드 | 취합 번거로움, 관리자 수동 집계         |
| 용병 모집 | 네이버 카페 | 외부 플랫폼 이탈, 연락 이중화           |
| 경기 기록 | 엑셀 / 없음 | 누적 관리 불가, 개인 스탯 없음          |
| 회비 정산 | 카카오뱅크  | 납부 현황 실시간 확인 불가              |

> **"아마추어 축구팀은 경기 자체보다 운영 행정에 더 많은 에너지를 쓰고 있다."**

---

## 3. 핵심 타겟

**Primary** — 서울·경기·인천 아마추어 축구·풋살 팀 운영진 (20~40대, 팀장·총무·주장)

**Secondary** — 일반 팀원 (공지 확인·투표·스탯 확인), 용병 플레이어

---

## 4. 핵심 가치 (USP)

> **"매주 반복되는 팀 운영 행정을 없앤다."**

| 구분      | 기존 플랫폼    | FC Flow                   |
| --------- | -------------- | ------------------------- |
| 핵심 초점 | 매칭·예약 중심 | **운영 자동화 중심**      |
| 타겟      | 개인 플레이어  | **팀 단위 운영자**        |
| 투표      | 없음           | **자동 생성 + 리마인드**  |
| 경기 기록 | 없음           | **누적 스탯 + MOM 투표**  |
| 용병      | 일부 지원      | **팀 내에서 원클릭 모집** |

## 6. 출시 전략 (Phase)

### Phase 1 — MOM (현재 개발 대상)

> 핵심 루프: **로그인 → 팀 가입 → 경기 투표 → 경기 기록**

| 카테고리  | 포함 기능                                                |
| --------- | -------------------------------------------------------- |
| 인증      | 이메일/패스워드 로그인·회원가입·로그아웃·토큰 갱신       |
| 프로필    | 온보딩 정보 입력, 매너 점수, 업적, 경기 통계             |
| 팀 관리   | 팀 생성·검색·가입·탈퇴·초대·팀원 목록·게시판·포지션 배정 |
| 경기 관리 | 경기 투표·오늘의 경기·경기 기록 입력·조회·MOM 투표       |

### Phase 2

- 소셜 로그인 전환 (카카오·구글·애플)
- 경기 매칭 (상대팀 모집·탐색)
- 용병 시스템 (용병 구함·용병 가능)
- FIFA 선수 프로필 카드 고도화

### Phase 3

- 구장 지도·예약 연동
- 회비 자동 정산·결제
- 리그 시스템

### Phase 4

- 프리미엄 구독·광고 연동
- 영상 분석

## 7. 도메인별 기능 요약

### 7-1. 인증 & 유저

**인증 방식**

- Phase 1: 이메일/패스워드 (JWT, RT Rotation, SecureStore)
- Phase 2: 소셜 로그인 전환 시 email·passwordHash 완전 삭제, `provider`+`providerId`로 계정 식별
- 상세: `docs/client-docs/specs/auth.md`

**유저 핵심 정책**

- **1인 1팀**: 동시에 하나의 팀에만 소속 가능
- **등번호**: User가 아닌 `ClubMember` 테이블 저장 (팀마다 다름)
- **Soft Delete**: 탈퇴 시 PII 즉시 파기, 경기 기록은 "탈퇴 사용자"로 익명화 보존

**온보딩 수집 항목**

이름(닉네임) / 출생연도 / 성별 / 주 포지션 / 주 발 / 경력 / 실력 / 선호 지역

**능력치 정책**

최초 입력: 본인 / 이후 수정: 관리자(주장·부주장)만 — 감독이 선수를 평가하는 UX

**매너 점수**

| 항목           | 내용                               |
| -------------- | ---------------------------------- |
| 초기값         | 100°C                              |
| 이용 제한      | 20°C 이하                          |
| 평가 항목      | 시간 준수 / 실력 / 태도 / 의사소통 |
| 정규 팀원 평가 | 팀 관리자가 관리                   |
| 매치전 선수    | 상대팀 선수도 평가 가능            |
| 용병           | 함께 뛴 팀의 선수들이 평가         |
| 자체전         | 매너 평가 없음                     |

상세: `docs/plans/auth-user.plan.md`

---

### 7-2. 클럽 (팀) 관리

**핵심 정책**

| 정책               | 내용                              |
| ------------------ | --------------------------------- |
| 1인 1팀            | 동시 중복 가입 불가               |
| 클럽 평점          | 소속 팀원 매너 점수 평균          |
| 모집 상태          | 최대 인원 도달 시 자동 마감       |
| 등번호             | 팀 내 unique 제약                 |
| 초대 코드 유효기간 | 7일, 만료 후 재발급 가능          |
| 강퇴 이력          | `ClubBanRecord` 저장, 재가입 제한 |

**주요 화면**

- 클럽 생성 (Funnel 3단계)
- 클럽 탭 (전적·이번 주 경기·주요 선수·추천 클럽)
- 클럽 검색 (이름 / 지역 / 내 주변 5km / 선호 지역)
- 가입 신청 관리 (중복 방지 Unique 제약, 신청 취소 지원)
- 팀원 목록·상세 (FIFA 카드 스타일, 능력치 애니메이션)
- 게시판 (공지·일반·문의, 삭제: 작성자 본인 + 주장)
- 권한 이전: 주장은 권한 이전 후에만 탈퇴 가능
- 팀 해체: 주장 요청 + **전원 동의** → 해체, 과거 경기 기록 "(해체한 팀)"으로 보존

**지역 데이터**

공공데이터포털 행정구역 코드 → Prisma seed.ts → API → Select Box

상세: `docs/plans/club.plan.md`

### 7-3. 경기 관리

**경기 상태 자동 전환** (시간 기반)

```
경기 전  →  경기 중  →  경기 후
(시작 시각 전)  (시작~종료)  (종료 시각 후)
```

**투표**

- 등록 권한: 관리자 전용
- 마감 후 변경 불가 (관리자 수동 처리만 예외)
- 포지션 배정과 연동: 투표 참석자 자동 로드 + 수동 추가

**포지션 배정**

- 매칭전: 쿼터별 포메이션·포지션 배정
- 자체전: 팀 나누기(N팀) + 쿼터별 배정
- 경기 중 실시간 동기화 (WebSocket/Polling)

**경기 기록**

- 등록·수정: 관리자 전용, 수정 시 변경 이력 저장
- MOM: 동점 시 공동 MOM, 중복 투표 방지 DB Unique 제약
- 활성화: 관리자가 기록 등록 시 푸시 알림 → 당일 자정 마감

**경기 상세 탭 구성**

- 경기 기록 (MOM 투표 포함)
- 댓글
- 영상 (유튜브 웹뷰)
- 상대팀 평가 (**매칭전만 표시**, 자체전 탭 숨김)

상세: `docs/plans/match.plan.md`

---

### 7-4. 경기 매칭 (Phase 2)

팀 단위로 상대팀을 모집·탐색하는 독립 탭.

**매칭 상태**: 모집 중 → 매칭 완료 (수락) / 만료 (날짜 경과 자동)

**매칭 성사 플로우**

1. 상대팀 관리자가 "매칭 신청"
2. 등록팀 관리자 수락/거절
3. 수락 시 양측 연락처(이름 + 휴대폰) **인앱 화면 표시**
4. 외부 앱(전화·문자)으로 세부 조율

상세: `docs/plans/matching.plan.md`

---

### 7-5. 용병 시스템 (Phase 2)

탑탭으로 "용병 구함" / "용병 가능" 구분.

**용병 구함** — 팀 관리자가 등록, 지원자 관리·수락·거절

**용병 가능** — 개인 플레이어 등록, 팀이 영입 신청

**매칭 중개**

- 팀 → 개인: 영입 신청
- 개인 → 팀: 입단 신청
- 수락 전 연락처 미노출, 수락 후 인앱 표시

**게시글 만료**: 경기(가능) 날짜 경과 시 자동 만료 + 수동 삭제 지원

**블랙리스트**: 매너 온도 20°C 이하 / 노쇼 누적 / 계정 이용 제한 시 게시글 등록·노출 제한

상세: `docs/plans/mercenary.plan.md`
